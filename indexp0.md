Markdown es un lenguaje de marcado ligero desarrollado en 2004 por John Gruber, junto con la colaboración de Aaron Swartz. Su propósito principal fue crear una forma de escribir documentos en texto plano que pudiera transformarse fácilmente a HTML, manteniendo su legibilidad incluso antes de convertirlos.

A diferencia de otros lenguajes de marcado más avanzados, Markdown se caracteriza por tener una sintaxis sencilla e intuitiva, lo que hace que redactar y leer contenido sea mucho más práctico. Gracias a esto, se volvió una herramienta muy popular en distintos contextos, como:

Documentación técnica (manuales, instructivos).

Blogs y foros que admiten Markdown, como WordPress o Reddit.

Aplicaciones de notas, por ejemplo Obsidian, Notion o Joplin.

Archivos README usados en repositorios de GitHub y GitLab.

Algunas de sus normas básicas son:

Encabezados (headers): se escriben con el símbolo #. Ejemplo:
# Título 1  <h1>,
## Título 2  <h2>

Cursivas: se logran colocando un asterisco o guion bajo al inicio y al final: *Texto* o _Texto_.

Negritas: se utilizan dos asteriscos o guiones bajos: **Texto** o __Texto__.

Listas:

Ordenadas: 1. Primer elemento

No ordenadas: - Elemento o * Elemento.

GGracias a estas ventajas, Markdown se ha convertido en una herramienta muy utilizada tanto en el desarrollo de software como en la creación de contenido digital.

Markup

Para comprender mejor qué es Markdown, primero es útil entender el concepto de Markup.

El término Markup (o lenguaje de marcado) se refiere a un sistema que permite dar estructura y formato a un texto mediante símbolos o etiquetas especiales. El ejemplo más conocido es HTML (HyperText Markup Language), que se emplea en la web para organizar documentos con títulos, párrafos, listas, imágenes, enlaces y otros elementos.

Desde esta perspectiva, Markdown puede considerarse una versión simplificada de los lenguajes de marcado tradicionales:

En HTML se requieren etiquetas más elaboradas, por ejemplo: <b>Texto en negrita</b>.

En Markdown, la sintaxis se vuelve más ligera, usando simplemente: Texto en negrita.

Git y GitHub

Git es un sistema de control de versiones distribuido desarrollado por Linus Torvalds en 2005. Su propósito es permitir a los programadores registrar los cambios en un proyecto, trabajar de manera colaborativa y manejar distintas versiones del código de forma estructurada.

GitHub, por su parte, es una plataforma en la nube construida sobre Git. Ofrece herramientas adicionales para la colaboración, administración de proyectos, seguimiento de errores y trabajo en equipo. Fue fundada en 2008 y hoy en día es propiedad de Microsoft.

Comandos básicos de Git

Estos son algunos de los comandos fundamentales para comenzar a usar Git:

git config --global user.name "TuNombre"
git config --global user.email "tuemail@ejemplo.com"

git init
git add .
git add archivo.txt

git commit -m "Primer commit"

git remote add origin (link del repositorio)

git push -u origin main
git push   # Este último se usa para enviar y guardar los cambios
